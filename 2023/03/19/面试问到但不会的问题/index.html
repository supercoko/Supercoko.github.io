Linux

TOP

top 命令是 Linux 系统常用命令之一，能够实时显示系统各个进程的资源占用情况，类似于 Windows 系统的任务管理器。以下是一些常用的 top 命令参数123：

  • -d 秒数：指定 top 命令每隔几秒更新。默认是 3 秒；
  • -b：使用批处理模式输出。一般和 -n 选项合用，用于把 top 命令重定向到文件中；
  • -n 次数：指定 top 命令执行的次数。一般和 -b 选项合用；
  • -p 进程PID：仅查看指定 ID 的进程；
  • -s：使 top 命令在安全模式中运行，避免在交互模式中出现错误；
  • -u 用户名：只监听某个用户的进程。

IFTOP 查看网络流量

iftop命令来查看Linux系统的网络流量和带宽。

Iftop是一种基于文本的网络流量监视器，它可以显示当前网络连接的实时流量。

  • 您可以使用以下命令安装iftop：sudo apt-get install iftop。
  • 使用以下命令启动iftop：sudo iftop。
  • 您可以使用以下命令来查看特定接口的流量：sudo iftop -i eth0。
  • 您可以使用以下命令来查看特定主机的流量：sudo iftop -F 192.168.1.1/24

iftop 命令的常用参数如下：

  • -n：禁用 DNS 反向解析。
  • -N：禁用 DNS 正向解析。
  • -p：指定要监视的端口。
  • -i：指定要监视的网络接口。
  • -B：指定要使用的带宽单位。
  • -F：指定要使用的过滤器。
  • -f：指定要使用的配置文件。

网络

HTTP1.0和HTTP1.1区别

HTTP1.1相对于HTTP1.0的主要改进有以下几点：

  1. 持久连接：HTTP1.1支持持久连接，可以在一个TCP连接上发送多个HTTP请求和响应，减少了TCP连接的建立和关闭的开销，提高了性能。
  2. 分块传输编码：HTTP1.1支持分块传输编码，可以将一个大的HTTP响应分成多个块进行传输，提高了传输效率。
  3. 虚拟主机：HTTP1.1支持虚拟主机，可以在一台服务器上托管多个域名，提高了服务器的利用率。
  4. 缓存控制：HTTP1.1支持更加灵活的缓存控制，可以通过Cache-Control和Expires等头部字段来控制缓存的行为。
  5. 更加严格的语法：HTTP1.1对HTTP1.0的语法进行了一些修改和完善，使得HTTP1.1更加严格和规范。

HTTP1.0和1.1应用场景

HTTP1.0和HTTP1.1的应用场景没有明显的区别，但是HTTP1.1相对于HTTP1.0有更好的性能和更加灵活的缓存控制，因此HTTP1.1更加适用于高性能的Web应用。

持久连接和非持久连接

HTTP持久连接和非持久连接的主要区别在于TCP连接的使用方式。非持久连接每次请求都需要建立一个新的TCP连接，请求结束后立即关闭TCP连接，而持久连接可以在一个TCP连接上发送多个HTTP请求和响应，减少了TCP连接的建立和关闭的开销，提高了性能。

  • 一个使用非持久连接的例子是，当您在浏览器中访问一个网站时，浏览器会为每个请求建立一个新的TCP连接，请求结束后立即关闭TCP连接。这种方式可以保证每个请求的独立性，但是会增加TCP连接的建立和关闭的开销，降低性能。

  • 一个使用持久连接的例子是，当您在浏览器中访问一个使用HTTP1.1协议的网站时，浏览器会在一个TCP连接上发送多个HTTP请求和响应，减少了TCP连接的建立和关闭的开销，提高了性能。

HTTP和HTTPS区别

HTTP和HTTPS都是超文本传输协议，但是HTTPS是具有安全性的SSL/TLS加密传输协议，信息是加密传输的。HTTP和HTTPS使用的是完全不同的连接方式，用的端口也不一样，前者是80，后者是443。

非对称加密

公钥加密的过程如下：

  1. 接收方生成一对密钥，一把是公钥，一把是私钥。
  2. 接收方将公钥发送给发送方。
  3. 发送方使用接收方的公钥对信息进行加密。
  4. 接收方使用自己的私钥对信息进行解密。

SSL/TLS

SSL（Secure Sockets Layer）和TLS（Transport Layer Security）是一种公钥加密技术，用于保护网络通信的安全。SSL/TLS协议可以保证信息的机密性、完整性和真实性，是HTTPS协议的核心。

证书

HTTPS证书是一种由数字证书认证机构（CA）颁发的数字证书，用于证明网站的身份。HTTPS证书包含了网站的公钥，以及证书的颁发机构、有效期等信息。当用户访问一个使用HTTPS协议的网站时，浏览器会向网站请求证书，然后验证证书的有效性，如果证书有效，则使用证书中的公钥对信息进行加密传输。

HTTPS一定安全吗

HTTPS协议可以保证信息的机密性和完整性，但是并不能保证信息的真实性，也就是说，HTTPS协议并不能保证网站的安全性。如果网站本身存在漏洞或者被攻击者篡改，那么HTTPS协议也无法保证信息的安全。

测试

app性能测试的指标

APP性能测试的指标通常包括：耗电量、耗流量、启动退出耗时、响应时延、流畅度、crash率、卡顿、CPU内存等12345。

这些指标可以帮助开发人员了解应用程序的性能，以便在应用程序发布之前进行必要的优化和调整。不同的应用程序可能会有不同的性能测试指标，具体的指标需要根据应用程序的特点和需求来确定12345。

1、内存：内存消耗测试节点的设计目标是为了让应用不占用过多的系统资源，且及时释放内存，保障整个系统的稳定性。当然关于内存测试，

  • 在这里我们需要引入几个概念：
    • 空闲状态、
      • 空闲状态指打开应用后，点击home键让应用后台运行，此时应用处于的状态叫做空闲
    • 中等规格、满规格。
    • 中等规格和满规格指的是对应用的操作时间的间隔长短不一，中等规格时间较长，满规格时间较短。
  • 内存测试中存在很多测试子项，清单如下：
    • 空闲状态下的应用内存消耗；
    • 中等规格状态下的应用内存消耗；
    • 满规格状态下的应用内存消耗；
    • 应用内存峰值；
    • 应用内存泄露；
    • 应用是否常驻内存；
    • 压力测试后的内存使用。
  1. CPU： 使用Android提供的view plncopy在CODE上查看代码片派生到我的代码片 adb shell dumpsys CPUinfo |grep packagename >/address/CPU.txt来获取； 使用top命令view plncopy在CODE上查看代码片派生到我的代码片 adbshell top |grep packagename>/address/CPU.txt来获取。
  2. 流量： 网络流量测试是针对大部分应用而言的，可能还有部分应用会关注网速、弱网之类的测试。 流量测试包括以下测试项：
    1. 应用首次启动流量提示；
    2. 应用后台连续运行2小时的流量值；
    3. 应用高负荷运行的流量峰值。
  3. 电量：
    1. 测试手机安装目标APK前后待机功耗无明显差异；
    2. 常见使用场景中能够正常进入待机，待机电流在正常范围内；
    3. 长时间连续使用应用无异常耗电现象。
  4. 启动速度：
    1. 第一类：首次启动–应用首次启动所花费的时间；
    2. 第二类：非首次启动–应用非首次启动所花费的时间；
    3. 第三类：应用界面切换–应用界面内切换所花费的时间。
  5. 滑动速度、界面切换速度
  6. 与服务器交互的网络速度

微信登录测试用例设计

  1. 功能测试
    1. 输入正确的用户名和密码，点击提交按钮，验证是否能正确登录。
    2. 输入错误的用户名或者密码,验证登录会失败，并且提示相应的错误信息。
    3. 登录成功后能否能否跳转到正确的页面
    4. 检查能否选择不同登录方式进行登录，如使用手机号登录、使用微信号登录或扫码登录。
    5. 记住用户名的功能
    6. 登陆失败后，不能记录密码的功能
    7. 密码是否非明文显示显示，使用星号圆点等符号代替。
    8. 有验证码时，还要考虑文字是否扭曲过度导致辨认难度大，考虑颜色、刷新或换一个按钮是否好用
    9. 登录页面中的注册、忘记密码，登出用另一帐号登陆等链接是否正确
    10. 输入密码的时候，大写键盘开启的时候要有提示信息。
    11. 什么都不输入，点击提交按钮，检查提示信息。
  2. 界面测试
    1. 布局是否合理，testbox和按钮是否整齐。
    2. testbox和按钮的长度，高度是否复合要求。
    3. 界面的设计风格是否与UI的设计风格统一。
    4. 界面中的文字简洁易懂，没有错别字。
  3. 性能测试
    1. 打开登录页面，需要的时间是否在需求要求的时间内。
    2. 输入正确的用户名和密码后，检查登录成功跳转到新页面的时间是否在需求要求的时间内。
    3. 模拟大量用户同时登陆，检查一定压力下能否正常登陆跳转。
  4. 安全性测试
    1. 登录成功后生成的Cookie，是否是httponly (否则容易被脚本盗取)。
    2. 用户名和密码是否通过加密的方式，发送给Web服务器。
    3. 用户名和密码的验证，应该是用服务器端验证， 而不能单单是在客户端用javascript 验证。
    4. 用户名和密码的输入框，应该屏蔽SQL注入攻击。
    5. 用户名和密码的的输入框，应该禁止输入脚本 （防止XSS攻击）。
    6. 防止暴力破解，检测是否有错误登陆的次数限制。
    7. 是否支持多用户在同一机器上登录。
    8. 同一用户能否在多台机器上登录。
  5. 兼容性测试
    1. 不同移动平台或PC环境下下能否显示正常且功能正常
    2. 同种平台下不同微信版本下能否显示正常且功能正常。
    3. 不同的分辨率下显示是否正常。
  6. 本地化测试
    1. 不同语言环境下，页面的显示是否正确。

面向对象和面向过程的区别

面向过程和面向对象是两种不同的编程思想。面向过程是一种以事件为中心的编程思想，编程的时候把解决问题的步骤分析出来，然后用函数把这些步骤实现，在一步一步的具体步骤中再按顺序调用函数。而面向对象是将构成问题的事物，分解成若干个对象，建立对象的目的不是为了完成一个步骤，而是为了描述某个事物在解决问题时所扮演的角色。面向对象具有三大特征：封装性、继承性和多态性，而面向过程没有继承性和多态性，并且面向过程的封装只是封装功能，而面向对象可以封装数据和功能
