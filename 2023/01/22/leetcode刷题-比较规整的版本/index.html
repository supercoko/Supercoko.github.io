19. 删除链表的倒数第 N 个结点

中等题

双指针

链表长度为 S

先让一个指正从头节点开始走n次, 此时其指向第 n+1 个节点

然后再让一个指针指向 头结点, 然后两个节点同步向下走, 直到快的节点(之前走过n步的节点)走到结尾None(Null) 即 第 S + 1个节点

慢节点和快节点恒定差n个节点, 此时慢节点指向 S + 1 - n 即 第 倒数第n个节点, 此时获取此节点前一个节点, 让其指向下一个节点即可, 此时可用虚拟节点来解决

while循环还利用了短路效应, 此时当fast节点为空时, 就不会进入fast.next从而引发报错

数学模拟

$v_a = 1 m/s, v_b = 2m/s$

当b停止时, 行驶了3s

a到3m处

b到6m处

此方法当链表中点有两个时, 会指向靠后的那一个, 如 有6个节点, 慢指针最后会指向第4个节点

567. 字符串的排列

动态规划也能做, 但这里采用滑动窗口

  • 内容

    • 右移 (扩张)
      • 如果字符出现在s1中, 则, 该字符频次+1
      • 当字符频次和s1中相同时, valid + 1, 代表这个字符已经出现够次数了
    • 左移 (收缩)
      • 如果长度等于s1的长度(要进行排列的字符串), 开始收缩
      • 当 valid数 和 s1 字符种类数量一致时, 代表找到
      • 如果左边字符在s1中, 则对应频次-1, 如果频次等于 s1中改字符频次, 则valid-1
    • 如果右边界小于左边界, 则让左边界等于当前右边界,进入下一轮循环

  • 复杂度

    • 时间复杂度

    • 空间复杂度

  • 代码

  • PS.
    • 在本题中, 其实可以用数组结构代替 Counter 和defaultdict 这两种哈希结构,
    • 限定为小写字母, 那就有26种情况
    • cnt,win = [0]*26
    • 遍历s1, 用assii码来代替字典中key, 即 cnt[0] 就是cnt[‘a’], 让 字符的ascii码 减去 ‘a’ , 其实就是自己做了下哈希映射
