剑指OFFER

排序

1. (简单)剑指 Offer 40. 最小的k个数

思路1: 直接排序,返回数组前k个

思路2: 利用快排序的特性, 每次执行后, 哨兵pivot左边为比其小的数, 题目要求是返回最小的k个数, 并不要求排序, 所以可以判断当哨兵数的索引位置来判断哨兵左边的有几个比其小的数, 当其等于k时,意味着此时的左边的数为k个最小的数.

时间复杂度为O(N), 空间复杂度为O(logN) (执行logN层)

2. (困难)剑指 Offer 41. 数据流中的中位数

思路1: 直接排序

思路2: 利用堆的特性解决

搜索与回溯

1. (简单)剑指 Offer 55 - I. 二叉树的深度

思路1: 用DFS遍历(后序遍历)

  • 递归执行, 每次返回最大的子树的深度 + 1
  • 结束条件为node为none, 即此处无节点, 返回0
  • 例如 只有一个root节点,root.left为0, root.right为0, 则返回 0+1 即深度只有1

时间复杂度 O(N)

空间复杂度 O(N) ,最差情况下(树退化为链表),递归深度为N, 递归的空间是要计算的, 好像是叫栈间空间

思路2: 用BFS遍历

  • 每遍历一层增加1
  • 利用队列先进先出的特性(理解错了, 不一定队列, 因为先左先右每影响)
    • 添加根节点
    • pop出来
    • 判断这个节点有没有左右子节点, 有哪个添加进去哪个
    • 此时队列里是一层的节点

2. (简单)剑指 Offer 55 - II. 平衡二叉树

思路1:求左右子树的深度, 如果任意左右子树深度之差<=1则为true

  • 先序, 先根后左右, 就是先判断上一层, 再判断下一层
  • 因为递归中返回的false不会终止上一层递归, 因此只能利用其返回值进行判断

时间复杂度 O(NLogN), 每层的时间复杂度约为O(N), 有N个节点则遍历层数近似为O(logN)层

空间复杂度 O(N), 存储n个递归的栈空间

思路2: 后序遍历,从底到根, 先左右,后根.

  • 返回值

    • 当左右子树的深度差<=1则返回当前子树深度

    • 当深度差>1时, 则返回-1

  • 终止条件:

    • 当节点为空时, 返回高度0
    • 当子树深度为-1时, 代表此子树不是平衡树, 因此剪枝, 直接返回-1

  • pyhton中需要定义一个全局?变量去存储递归产生的结果值

4. (简单) 剑指 Offer 68 - I. 二叉搜索树的最近公共祖先

这题居然是简单吗???!

祖先定义:

  • p在root的子树中, 或这p=root, 则root为p的祖先

最近公共祖先定义:

  • root为p,q某公共祖先
  • 其左, 右子节点都不是p,q公共祖先, 则称root为最近的公共祖先

若root为p,q 最近公共祖先

  • p,q 在root 子树, 且分列root的不同侧
  • p=root, 且q在root子树
  • q=root, 且p钻root子树

此树为二叉搜索树, 左树小, 右树大

思路1: 迭代

  • 节点为空则跳出
  • p,q都在右子树, 则遍历节点右子树
  • 都在左子树则遍历左子树
  • 否则, 说明找到了公共祖先root

时间O(N)

空间O(1)

思路2: 递归

时间O(N) 最优为logN(满二叉树), 最差为O树退化为链表

空间O(N), 树退化为来拿表, 迭代深度为N(深度等于节点数)

1. (简单)剑指 Offer 65. 不用加减乘除做加法

简单个锤子

无进位和 $n = a \oplus b$

进位值 $c= a & b$ << 1 进位后左移动一位

s = a + b => s = n + c

循环求n和c, 直到 c = 0 此时 s = n

现在对补码还是有点懵, 总结下就是补码可以做到加减法一致

时空间复杂度皆为 O(1)

python的补码机制比较特殊, 输出负数时会输出 负号 + 原码

正数的原码, 反码, 补码一致

负数的反码为原码按位取反(除符号位), 补码是反码+1 (负数的符号位是1)

(负数)补码变原码 为 符号位不变 按位取反 再加1

python中的按位取反, (计算机都是用补码进行操作的)

python中的数默认是 4字节 32 bit

python中没有位数限制….是题目要求的不会超过32位…搞错了 尴尬

PS. 只有运算过程按照补码来, 直接与结果还是按照原码??

3. (中等)剑指 Offer 14- I. 剪绳子

Python 中常见有三种幂计算函数： ** 和 pow() 的时间复杂度均为 O(log a) ；而 math.pow() 始终调用 C 库的 pow() 函数，其执行浮点取幂，时间复杂度为 O(1)

算数集合平均值不等式 等号成立时, 当且仅当 $n_1= n_2 = …=n_{a}$

$\frac{n_1+ n_2 + …+n_{a}}{a} \geq \sqrt[a]{n_1n_2…n+a}$

纯数学推导了属于是

latex用不太熟, 手写一遍吧

贪心算法!!!

由上可知

  • 最优的分段大小为3, 次优的为2
  • 利用dp存储当前数能得到的值
  • dp[1] = 1
  • dp[2] = 2
  • dp[3] = 3
  • dp[4] = max(dp[2]*2, dp[1]*3)
  • dp[i] = max(dp[i-2]*2, dp[i-3]*3)

5. (简单)剑指 Offer 62. 圆圈中最后剩下的数字

简单个👻啊

用链表解决时间复杂度过于复杂

O(NM)

动态规划解决

约瑟夫环

!!!

(A + B) % C = (A % C + B % C ) % C

被删除的数字为 (m - 1)%n

有n个数

0, 1, 2, … n-1

第m个数为

m - 1

又因为如果m-1大于n时, 会从头开始

所以删除的数字为 (m-1)% n

t = m % n

(m-1)%n = (m%n - 1%n)%n = (t - 1%n)%n

因为等于小于的数取余数等于其吱声

e.g. 1 % 2 = 1, 3%4=3

上式 = (t - 1) % n = t - 1

因此可得, t - 1 为被删除的数

t,t+1,t+2,…,0,1,…,t−3,t−2

[n - 1, m] 问题 -> [n , m] 问题删除后
0 -> t + 0
1 -> t + 1
… ->
n-2 -> t-2

设[n - 1, m] 问题, 某数字为x,

x -> (x+t)%n

后面就晕了

…

因为 n-1 里的 x 等于 n 里的 (x+t)%n

所以 n-1的解 f(n-1) 对应 n 里的 (f(n-1)+t)%n
