charles简介

charles是什么

基于HTTP协议的代理服务器, 通过成为电脑或浏览器的代理, 截取请求和请求结果达到分析抓包的目的

工作原理

前置步骤

  • 运行charles并配置代理
  • 在客户端上配置代理

步骤

  1. 客户端发送请求

  2. charles 接收在发送给服务端

  3. 服务端返回请求结果给charles

  4. charles转发给客户端

能做什么

  • 支持HTTP和HTTPS代理
  • 支持流量控制
  • 支持接口并发请求
  • 支持重发网络请求
  • 支持断点调试

优点

  1. 支持按照域名和接口查看报文
  2. 反向代理
  3. 网络限速可以选择网络类型
  4. AMF协议

抓包分析

人事管理系统中修改员工信息时, 如果员工姓名超长, 会产生一下两个问题

  • 提示 “系统繁忙”
    • 后台
  • 显示两次提示信息
    • 后台只发送一次, 所以是前端问题

修改请求和响应

MAP (长期的将某些请求重定向到另一个网络地址或本地文件)

Tools -> Map Remote/Map Local

  • Map Remote
    • 从源地址 到 目的地址
  • Map Local
    • 从源地址 到 本地目标文件
  • save Response 可以保存请求结果到本地

Rewrite (对网络请求做一些正则替换)

修改请求或返回结果Body或者Header内容, 支持正则表达式

注意, 如果和map remote 配合使用, 需要勾选 Preserve host in header fields

BreakPoints(临时替换一次)

  • 场景一 编辑前端传输

    • 前端对一些输入做了限制, 为了验证后端是否也做了限制
      • 比如说某个信息输入不能为空, 前端不输入东西, 不会向后端发送, 想要测试后端部分是否能判断空信息不允许输入, 可以通过断点测试修改前端传送的数据模拟网络丢失数据的情况, 看后端做的回应是否正确.

  • 场景二 边界后端返回

    • 看后端返回信息, 前端的回应是否正确
