面向对象的三大特性

  1. 封装 - 将属性 方法封装到一个抽象的类中
  2. 继承 - 实现代码的重用, 相同代码不需要重复编写
  3. 多态 - 不同对象调用相同的方法, 产生不同的执行结果, 增加代码灵活度

GIL Global Interpreter Lock

全局解释器锁

解释器用于同步线程的一种机制, 同一时刻只能运行一个线程,即使有多核CPU

python GIL, 和对多线程的影响,, 编写一个多线程抓取网页的程序, 说明多线程相比单线程是否比单线程性能有提升?

  1. GIL和python没有关系, 由于Cpython, GIL难以移除

  2. GIL 全局解释器锁. 每个线程在执行的过程中需要先获取GIL, 保证同一时刻只有一个线程可以执行代码

  3. 线程释放GIL锁: 在IO操作等可能引起阻塞的system call之前, 可以暂时释放GIL, 但在执行完毕后, 必须重新获取GIL. Python3中使用计时器(执行时间达到阈值后, 当前线程释放GIL) 或者python2 tickets技术达到100

  4. pyhton 使用多进程是可以利用多核COU资源的

  5. 多线程爬取比单线程性能有提升, 因为遇到IO组设会自动释放GiL锁

多线程(python)实际上是”假的”并行, 实际同一时间段只有一个线程在执行, 因为GIL

计算密集型:进程优先

IO密集型: 线程, 协程

具有IO密集型的的适合多线程(python)

解决方案

  • 换解释器
  • 换语言

深浅拷贝

引用(指向内存某处)

浅拷贝 会指向一个新的内存快, 但浅拷贝列表内的元素还是指向原来的地方

不可变元素浅拷贝,深拷贝, 拷贝的只是指向(引用)

如果元组有指向的可变元素, 则深拷贝是会正常运行

总结

如果用copy.copy copy.deepcopy对一个全部是不可变类型的数据进行拷贝, 那么他们结果相同, 都是已引用指向

如果拷贝的是一个拥有不可变类型的数据, 即使元组是最顶层, 那么deepcopy依然是深拷贝, 而copy.copy还是指向

拷贝的其他方式

  • d = c[:]与 d = copy.copy(c) 都是浅拷贝

  • 字典的拷贝属于浅拷贝
