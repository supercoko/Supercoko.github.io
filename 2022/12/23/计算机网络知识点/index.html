1.5 计算机网路的性能指标

1 速率

1 KB = 1024 B = 2^10 B

  • 小习题 100MB 速率为100Mbps 传输完成需要多少秒?
    • 100MB = 100 * 2^20 B = 100 * 2^23 bit
    • 100Mbps = 100 * 10 ^ 6 bit/s
    • t = 100 * 2^23/10^8 = 8.388608

2 带宽

在模拟信号系统中

  • 信号包含的各种不同频率成分所占据的频率范围
  • 单位Hz kHz MHz GHz
  • 带宽3.1kHz (300 Hz-3.4kHz)

带宽在计算机网络中的意义

  • 用来表示网络的通信线路所能传输数据的能力, 因此网络带宽表示在单位时间内从网络的某一点到另一点所能通过的 最高数据率

  • 单位 b/s

  • 带宽的两种表述指甲你有这密切联系, 一条通信线路的频率宽度约款, 栖所传输数据的最高数据绿也越高

3 吞吐量

  • 单位时间内通过某个网络,信道或者接口的数据量
  • 常被用于显示世界中的网络的一种测量,以便知道实际上有多少数据量能通过网络
  • 受网络带宽或额定速率的限制

4 时延

网络时延 = 发送时延(传输时延) + 传播时延(考虑介质) + 处理时延(不方便计算) + 排队时延(没有说)

5 时延带宽积

  • 传播时延 x 带宽

  • 若连续发送数据,则在发送的第一个比特即将到达终点时, 发送短就已经发送了时延带宽积个比特

  • 链路的时延带宽积又称为以比特为单位的链路长度

6 往返时间 RTT Round-Trip Time

7 利用率

信道利用率

  • 表示某信道有百分之几的时间是被利用的(有数据通过)

网络利用率

  • 全网络的信道利用率低加权平均

根据排队理论, 某信道利用率增大时, 该信道引起的时延也会迅速增加, 因此, 信道利用率不是越高越好

  • D_0表示网络空闲的时延, D表示网络当前的时延, 在适当的假定条件下, 可以利用简单公式表示D,D_0和利用率U之间的关系.
  • D = D_0/1-U
    • 达到50% 时延加倍
    • 超过50% 急剧增加
    • 接近100% 无穷大
    • 因此一般不会超过50%, 通常会动态调整吞吐量来确保既不浪费也不过多增大时延

8 丢包率

  • 丢包的分组数量与总分组数量的比率
  • 可以分为
    • 接口丢包率
    • 结点丢包率
    • 链路丢包率
    • 路径丢包率
    • 网络丢包率
    • 等等
  • 两种导致原因
    • 分组在传输过程中出现误码, 被结点丢弃
    • 分组在达到一台队列已满的分组交换机时被丢弃, 通信量较大时可能造成网络拥塞
  • 反应了网络的拥塞情况
    • 无拥塞时, 0
    • 轻度拥塞, 1~4%
    • 严重拥塞, 5~15%

4 计算机网络体系结构专用术语

实体

实体

  • 任何可发送或接受信息的==硬件==或==软件进程==

对等实体

  • 收发双方相同层次中的实体

协议

协议

  • 控制两个对等实体进行逻辑通信的规则的集合

协议的三要素

  • 语法
    • 定义所交换信息的格式
  • 语义
    • 定义收发双方所要完成的操作
  • 同步
    • 定义收发双方的时序关系
服务

  • 在协议的控制下,两个对等实体间的逻辑通信是的本层能向上一层提供服务

  • 实现本层协议还需要使用下面一层所提供的服务

  • 协议时水平的, 服务时垂直的

  • 实体看的见相邻下层所提供的服务, 但比不知道实现该服务的具体协议, 也就是说下面的协议对上面的实体时透明的.

  • 服务访问点 在同一系统中相邻的实体交换信息的逻辑接口,用于区分不同的服务类型

    • 数据链路层的服务访问点为帧的 “类型” 字段
    • 网络层的服务访问点为IP数据报手部中的“协议字段”
    • 运输层的服务访问点为端口号

  • 服务原语 上层使用下层所提供的服务必须通过与下层交换一些命令

  • 协议数据单元 PDU 对等层次之间传送的数据包称为该层的协议数据单元

  • 服务数据单元 SDU 同一系统内, 层与层之间交换的数据包

  • 多个SDU可以合并成一个pdu, 一个sdu 也可以划分为几个pdu

透明 下一层对上一层是透明的, 具体理解是上一层不用知道下一层是如何传输的数据, 直接使用即可

第三章 数据链路层

3.1 概述

数据链路层在网络体系结构中所处的地位

链路 link 从一个结点到相邻结点的一段物理线路, 中间没有任何其他交换结点

数据链路 Data Link 实现通信协议的硬件和软件驾到链路上, 构成数据链路

数据链路层以==帧==为单位传输和处理数据

数据链路层的三个问题 (点对点)

  1. 封装成帧
  2. 差错检测
  3. 可靠传输
    1. 误码是不可避免的, 但若能实现发送方发什么接受方就能收到什么, 就称为可靠传输

使用广播信道的数据链路层

  • 共享式以太网媒体介入控制协议 CSMA/CD
  • 802.11局域网的媒体介入控制协议 CSMA/CA

数据链路层的互联设别

  • 网桥和交换剂的工作原理
  • 集线器 与 交换机的区别

3.2 封装成帧

封装成帧是指数据链路层给上层交付的协议数据单元添加帧头和帧尾

  • 帧头和帧尾中包含有重要信息
  • 帧头和帧尾作用之一帧定界
  • 并不是每一种数据链路层协议都包含帧定界标志

透明传输

  • 指数据链路层对上层交付的传输数据没有任何限制, 就好像数据链路层不存在于一样
    • 不能不允许数据中存在帧定界标志, 否则不能成为透明传输, 一般解决方案为对数据进行扫描并在其前添加转移字符
  • 面向字节的物理链路使用字节填(字符)的方式实现透明传输
  • 面向比特的物理两路使用比特填充方法实现比特传输

为了提高帧的传输效率, 应当使帧的数据部分长度尽可能大些

考虑到差错控制等因素, 每种数据链路层协议都规定了帧的数据部分长度上限, MTU Maximum Transfer Unit 最大传送段元

3.3 差错检测

比特差错 : 0->1 1->0

一定时间内, 传输错误的比特占所传输比特中数的比率成为 BER Bit Error Rate 误码率

使用差错检测码检测是否出现比特差错

检测方法

  • 奇偶校验 (漏检率较高->检测不出误码) 一般不采用
    • 在待发送的数据后面==添加1位奇偶校验位==,使整个数据, 包括所添加的校验位在内 , 中1 的个数为奇数 (奇校验), 或偶数(偶校验)

3.6.2 媒体接入控制 - 静态划分信道

信道复用

  • 复用 multiplexing 是通信技术中的一个重要概念, 复用就是通过一条物理线路同时传输多路用户信号
  • 当网络中传输媒体的传输容量大于多条单一信道传输的总通信量时, 可利用复用技术在一条物理线路上建立多条通信信道来充分利用传输媒体的带宽
  • 频分复用FDM
  • 时分复用TDM
  • 波分复用WDM
    • 每路2.5 Gb/s 8路就是 20 Gb/s
  • 码分复用CDM
    • CDM 是另一种共享信道的办法, 由于该技术主要用于多址接入, 码分多址 CDMA Code Division Multiple Access
    • 同理, FDM和TDM通用可用于多址接入, 相应名词为 频分多址 FDMA和 时分多址TDMA]
    • 本课程不严格区分多址和复用
      • 复用是将单一媒体的频道支援划分为很多子信道, 这些子信道相互独立互不干扰. 从媒体的整体频带资源来看, 每个子信道只占用概媒体频带资源的一部分
      • 多址 (多点接入) 处理的是动态分配信道给用户. 这在用户仅仅暂时性地占用信道的应用中是必须的, 而所有的移动通信系统基本都属于这种情况, 相反, 信道永久性地分配给用户的应用中, 多址是不需要的 (对于无线广播和电视广播站)
      • 一定程度上 FDMA, TDMA 和CDMA可以看成是FDM , TDM 和CDM的应用
    • 与FDM和TDM不同, CDM的每一个用户可以在同样的时间使用同样的频带进行通信
    • 由于各用户使用经过特殊挑选的不同码型, 因此各用户互不干扰
    • CDM 最初用于军事通信 抗干扰强, 频谱类似白噪声不易发现
    • CDMA 每一个比特时间在划分为m个短间隔, 成为 码片 Chip , 通常m的值为64或128, 为简单期间, 举例中m为8
    • 使用CDMA的每一个站被指派唯一一个m bit码片序列 Chip Sequence
      • 发送比特1 就发送自己的m bit码片序列
      • 发送比特0 就发送自己的m bit码片序列的二进制反码
      • 为了方便 0 写为 -1, 1 写为 +1
      • 直接序列扩频 DSSS
    • 码片序列挑选原则:
      • 每个站的码片必须各不相同, 实际采用伪随机码序列
      • 每个站的码片序列必须相互正交 规格化内积为0

3.6.4 媒体接入控制 - 动态接入控制 - 随机接入

载波监听多址接入/碰撞避免 CSMA/CA Carrier Sense Multiple Access/Coillision Avoidance

  • 在无线局域网中, 仍可使用载波监听多址接入CSMA, 即发送帧之前先对传输媒体进行载波监听. 若发现有其他站在发送帧, 就推迟发送以避免碰撞

  • 在无线局域网中, 无法使用碰撞检测CD, 原因:

    • 无线信号强度动态范围很大, 接收信号往往远小于发送信号, 如果要在无线网卡上实现碰撞检测CD, 对硬件的要求非常高
    • 即使能够在硬件上实现, 由于无线点播传播的特殊性(存在隐蔽站问题), 进行碰撞检测的意义也不大

  • 802.11无线局域网使用CSMA/CA, 在CSMA基础上增加了一个碰撞避免CA功能

  • 由于不可避免所有碰撞, 且误码率较高, 802.11采用了数据链路层确认机制 停止等待协议

  • 802.11 MAC层标准定义了两种不同的媒体接入方式

    • 分布式协调功能 DCF distributed Coordination Function. 在DCF下, 没有中心控制焊点, 每个站点使用CSMA/CA协议, 通过中共信道来获取发送权, 这是802.11定义的默认方式
    • 点协调功能PCF Point Coordination Function PCF方式使用集中控制接入算法, 一般在接入点AP Access Point 实现集中控制, 是802.11定义的可徐昂方式,实际中较少使用

帧间间隔 IFS InterFrame Space

  • 802.11 标准规定, 所有站点必须在持续检测到信道空闲一段指定时间才能发送帧, 这段时间成为IFS
  • 帧间间隔长短取决于发送帧的类型
    • 高优先级所需等待时间短
    • 低优先级则长, 高优先级发送到信道上, 信道会变为 忙 状态, 低优先级则推迟发送, 减少发生碰撞机会
  • 常用帧间间隔
    • 短帧间间隔 SIFS 28us, 最短的帧间间隔, 用来隔开属于一次对话的各帧, 一个站点应当能够在这段时间内从发送方式切换到接收方式,
      • ACK,CTS. 有过长的MAC帧分片后的数据帧, 所有回答AP探寻的帧, 和在pcf方式中接入点AP发送的任何帧.
    • DCF帧间间隔 DIFS 128us, 不短帧间间隔SIFS长很多, DCF方式发送数据帧和管理帧

CSMA/CA协议的工作原理

  • 当站点检测到信道是空闲的, 并且发送数据帧不是成功发送完上一个数据帧之后立即连续发送的数据帧,则不适用退避算法
  • 以下必须使用退避算法
    • 在发送数据帧之前检测到信道处于忙状态时.
    • 在每一次重传一个数据帧时
    • 在每一次成功发送后要连续发送下一个帧时, (避免一个站点长时间占用信道)

CSMA/CA 退避算法

  • 执行退避算法时, 站点为退避及时其设置一个随机退避时间
    • 退避计数器减小到0就发送数据
    • 为减小到0, 而信道又转变为忙状态, 这是就冻结退避计时器树脂, 重新等待信道变为空闲, 再经过时间DIFS后, 继续启动退避计时器
  • 进行第i次退避时, 退避时间在时隙编号 0-2^(2+i)-1 中速记选一个, 然后乘以基本退避时间(时隙长度). 这样做是为了使不同站点选择相同退避时间的概率减少, 当其编号到达255 即第6次退避, 就不再增加

CSMA/CA的信道预约和虚拟载波监听

  • 为尽可能减少碰撞发生概率, 和降低碰撞影响, 802.11允许要发送数据的站点对信道进行预约
    • 源站发送数据帧之前发送一个短的控制帧, RTS 请求发送 Request To Sent, 包括源地址, 目的地址, 这次通信(包含相应确认帧)所需的时间
    • 若目的站正确接收RTS 且空闲, 则发送一个响应控制帧 成为允许发送CTS Clear To Send 它也包括这次通信所需的持续时间(复制自RTS)
    • 源站收到CTS后, 再等待一段SIFS后就可发送其数据帧
    • 若目的站正确收到了源站发来的数据帧, 再等待时间SIFS后, 就向源站发送确认帧ACK
  • 除源站和目的站一卫的其他各站, 在收到CTS帧 或数据帧 后推迟接入到无线局域网中, 这样可以保证源站和目的站之间的通信不会收到其他站的干扰
  • 如果RTS发生碰撞, 源站收不到CTS, 则执行退避算法重传RTS
  • RTS和CTS帧很短,碰撞概率和开销很小, 用很小的代价对信道进行预约往往很值得. 802.11提供3种用户选择
    • 使用RTS和CTS
    • 不适用RTS和CTS
    • 数据帧长度超过一定数值使用RTS 和 CTS
  • 除RTS,CTS, 数据帧也要写到需要时间, 这称为802.11的虚拟载波监听机制
  • 由于虚拟载波监听机制, 站点只要见听到RTS帧, CTS帧或数据帧,就能知道展会用时间, 不需要监听到信道上的信号, 因此虚拟载波监听机制能减少隐蔽站带来的碰撞问题

3.10 以太网交换机生成树协议 STP (具体原理超出大纲)

  • 添加冗余链路可以提高以太网可靠性
  • 形成网络环路带来的问题(冗余链路导致)
    • 广播风暴
      • 大量小号网络资源, 使得网络无法正常转发其他数据帧
    • 主机收到重复的广播帧
      • 大量小号主机资源
    • 交换机的帧交换表震荡(漂移)
      • 因为形成环状发送广播时会反向收到自己发出的帧, 从未导致正确的转发表被覆盖掉
  • 以太网交换机使用生成树协议 STP Spanning Tree Protocol, 可以在正价冗余链路来提高可靠性的同时又避免网络环路带来的个钟问题
    • 交换机能自动计算并构建一个逻辑上没有环路的玩过, 逻辑拓扑结构必须是树型的(无逻辑环路)
    • 最终生成的树型逻辑拓扑要确保联通整个网络
    • 当发生变化时, 交换机进行生成树的重新计算

4.6.2 路由信息协议RIP的基本工作原理

  • Routing Information Protocol是内部网关协议IGP中最先得到广泛使用的协议之一, RFC1058
  • RIP要求自治系统AS内的每一个路由器都要维护从它自己到AS内其他每一个网络的记录. 这是一组距离, 称为距离向量D-V Distance-Vector.
  • RIP使用跳数 Hop Count 作为度量Metric 来衡量到达目的的网络距离.
    • 路由器到直连网络的距离定义为1
    • 路由器到非直连网络的距离定义为所经过的路由器数加1
    • 一条路径最多包含15个路由器, 距离等于16时相当于不可达, RIP只适用于小型互联网
  • RIP认为好路由就是距离短路由
  • 当有多条距离相等的路由时, 进行等价负载均衡
  • RIP包含一下三个要点
    • 和水交换信息 仅和相邻路由器交换信息
    • 交换什么信息 自己的路由表
    • 何时交换信息 周期性交换 每30s
  • RIP 存在 坏消息传播的慢的问题
    • 又称为路由环路或距离无穷计数问题, 距离向量算法的一个固有问题, 可以采取多种措施减少发生概率或减小危害
      • 限制最大路径距离为15
      • 当路由表发生变化是立即发送更新保温, 而不仅是周期性发送
      • 让路由器记录收到某特定路由信息的接口, 而不让同意路由信息再通过此接口向反向传送 水平分割

4.6.3 开放最短路径优先OSPF的基本工作原理

  • OSPF Open Shortest Path First是为克服RIP缺点
    • 采用Dijkstra 提出的最短路径算法SPF
  • OSPF基于链路状态, 不像RIP是基于距离向量的
  • 算法上保证不会产生回路
  • 不限制, 更新效率高, 收敛速度快
  • OSPF有以下五种分组类型
    • 问候 hello 分组 用来发现和维护邻居路由器的可达性
    • 数据库描述 Database Description 分组 向邻居路由器给出自己的链路状态数据库中的所有链路状态项目的摘要信息
    • 链路状态请求 Link State Request 分组 向邻居路由器请求发送牟星链路状态项目的详细信息
    • 链路状态更新 Link State Update 分组 路由器使用这种分组将其链路状态进行洪泛发送, 即用洪泛法对全网更新链路状态
    • 链路状态确认 Link State Acknowledge 分组 对链路状态更新分组的确认分组
  • OSPF的基本工作过程
  • OSPF在多点接入网络中路由器邻居关系的建立
  • 为了能用于规模很大网络, ospf把一个自制系统再划分为若干个更小范围, 叫做Area 区域
    • 区域内路由器 IR internal router
    • 区域边界路由器 ABR area border router
    • 主干路由器 BBR backbone router
    • 自治系统边界路由器 ASBR AS border router

4.6.4 边界网关协议BGP的基本工作原理

  • BGP-4 四种报文
    • OPEN 打开报文 用来与相邻的另一个bgp发言人建立关系, 使通信初始化
    • UPDATE 更新报文 通告某一路由的信息, 以及列出要撤销的多条路由
    • KEEPALIVE 保活报文 用来周期性地证实邻站的连通性
    • NOTIFICATION 通知报文 用来发送检测到的差错

4.7 IPv4数据报首部格式

  • 版本

    • 4 bit, 表示IP协议版本
    • 通信双方版版本要一直

  • 首部长度

    • 4 bit 表示IP数据包首部的长度,该字段取值一4字节为单位
    • 最小十进制取值为5, 表示IP数据包首部只有20字节固定部分
    • 最大十进制取值为15, 表示IP数据包首部包含20字节固定部分和最大40字节可变部分

  • 可选字段

    • 1字节到40字节不等,用来支持排错, 测量及安全等措施
    • 可选字段年增加了IP数据包的功能, 但同时也使IP数据包的首部长度称为可变的, 这增加了每个路由器处理IP数据包的开销, 实际很少使用

  • 填充字段, 确保首部长度为4字节的整数倍,使用0进行填充

  • 区分服务

    • 8bit 用来获得更好服务
    • 只有使用区分服务时才使用该字段

  • 总长度

    • 16bit 表示IP数据包的总长度 首部+数据载荷
    • 最大取值为10进制的65535 2^16-1

  • 标识

    • 16bit 属于同一个数据包的个分片数据包应具有相同标识
    • IP维持一个计数器,没产生一个数据包,计数器加1, 并赋值给标识字段

  • 标志

    • 3bit
      • DF位
        • 1 不允许分片
        • 0 允许分片
      • MF位
        • 1 后面还有分片
        • 0 此为最后分片
      • 保留位
        • 必须为0

  • 片偏移

    • 13比特 直出分片数据包的数据载荷部分偏移其在原数据包的位置有多少个单位
    • 片偏移以8字节为单位

  • 生存时间TTL

    • 8 bit 最初以秒为单位, 最大生存周期为255秒, 当路由器转发IP数据报时, 将IP数据报首部中的该字段的值减去IP数据包在本路由器上所耗费的时间, 若部位0就转发, 否则丢弃
    • 现在以跳数为单外,路由器转发IP数据报是, 将IP数据报首部中的该字段的值减1, 若部位0就转发, 否则丢弃
    • 防止环路

  • 协议

    • 8 bit 志明哥IPv4数据报的数据部分是何种数据单元
    • 常用的一些协议和响应协议字段值

  • 首部校验和

    • 16 bit 用来检测首部在传输过程中是否出现差错. 比CRC校验码简单, 称为因特网校验和
    • 每经过一个路由器都要检测检验和
    • 由于IP层本身不提供可靠传输服务. 因此IPv6中不再计算

  • 源IP地址和目的IP地址

    • 各占 32 bit

4.7 网际层控制报文协议 ICMP

  • 为了更有效地转发IP数据报和提高交付成功的机会, 再网际层使用了网际控制报文ICMP Internet Control Message Protocol
  • 主机或路由使用ICMP来发送差错报告报文和询问报文
  • ICMP报文被封装在IP数据报中发送
  • 以下五种ICMP差错报告报文
    • 终点不可达
      • 当路由器或主机不能交付数据报时, 就向源点发送重点不可达报文.具体可再根据ICMP的代码字段细分为目的网络不可达, 目的主机不可达, 目的端口不可达, 目的网络未知,目的主机位置等13种错误
    • 源点抑制
      • 当路由器或主机由于拥塞而丢弃数据报时, 就向源点发送源点抑制报文, 使源点知道应当把数据报的发送速率放慢
    • 时间超过
      • 当路由器收到一个目的IP地址不是自己的IP数据包, 会将其生存时间TTL字段的时间减1. 若结果部位0, 则将该IP数据包转发出去. 若结果为0, 除丢弃该IP数据包外, 还要向源点发送时间超过报文
      • 另外,当终点在预先规定的时间内不能收到一个数据包的全部数据报片时, 就把已收到的数据报片丢弃,也会发送时间超过报文
    • 参数问题
      • IP首部中的校验和字段发现误码就丢弃, 并向源点发送参数问题报文
    • 改变路由 重定向
      • 路由器把改变路由报文发送给主机, 让主机知道下次应将数据报文发送给另外的路由器
  • 以下不应发送ICMP
    • 对ICMP差错报告文不再发送ICMP差错报告文
    • 对第一个分片的数据报片的所有后续数据报片都不发送ICMP差错报告文
    • 具有多播地址的数据报怒发送ICMP差错报告文
    • 对具有特殊地址的数据报不发送ICMP差错报告文
  • 常见ICMP询问报文
    • 回送请求和回答
      • ICMP回送请求报文是有主机/路由向一个特定目的主机发出询问, 收到此报文的主机必须给源主机或路由器发送ICMP回送回答报文, 这种询问报文用来测试目的站是否可达及了解其有关状态
    • 时间戳请求和回答
      • ICMP时间戳请求报文是请某个主机或路由器回答当前时间, 次询问报文用来进行时钟同步和测量时间
  • ICMP应用举例
    • 分组网间探测PING Packet InterNet Groper
      • 测试主机或路由器间的连通性
      • 应用层直接使用网际层的ICMP (没有通过运输层的TCP或UDP)
      • 使用ICMP回送请求和回答报文
    • 跟踪路由 traceroute
      • 用来测试IP数据包从源主机到达目的主机要经过哪些路由器
      • win
        • tracert命令
        • 应用层直接使用网际层ICMP
        • 使用了ICMP回送亲够和回答报文以及差错报告报文
      • Unix
        • traceroute命令
        • 在运输层使用UDP
        • 仅使用ICMP差错报告报文
      • 实现原理
        • 通过设置不同TTL的ICMP回送请求报文来判断
        • 例如, 第一个路由器发送TTL=1, 第二个路由发送TTL=2…

虚拟专用网VPN Virtual Private Network

  • 同一机构内不同部门的内部网络所构成的虚拟专用网VPN又被称为内联网VPN

  • 外地员工访问公司内部, 运行VPN建立和公司主机之间的VPN隧道, 这种vpn被称为远程接入vpn
  • IP隧道技术

  • 环节IPv4地址空间即将耗尽的问题

第五章 运输层

5.1 运输层概述

  • 物理层, 数据链路层以及网络层共同解决了将主机通过易购网络互联起来所面临的问题, 实现了主机到主机的通信
  • 实际网络中通讯的主体是位于通信两端主机中的进程
  • 如何为运行在不同主机上的应用进程提供直接的通信服务是运输层的任务, 运输层协议有被称为端到端协议
  • 运输层直接为应用进程间的逻辑通信提供服务
    • 运输层向高层用户屏蔽了下面网络核心的细节(网络拓扑, 路由选择协议等), 使应用进程看见的好像是两个运输层实体之间有调端到端的逻辑通信信道
    • 根据应用需求的不同,因特网的运输层为应用层提供了两中不同的运输协议, 即面向连接的TCP和无连接的UDP

5.2 运输层端口号, 复用与分用的概念

  • 进程使用进程标识符 PID 来标志
  • 不同操作系统使用不同格式的进程标识符
  • 因此必须使用同一的方法来对TCP/IP 体系的应用进程进行标识
  • TCP/IP体系的运输层使用端口号来区分应用层的不同应用进程
    • 端口号16 bit, 因此取值 0 - 65535
      • 熟知端口号: 0-1023 IANA指派给了TCP/IP体系中最重要的一些应用协议
        • FTP 21/20
        • HTTP 80
        • HTTPS 443
        • DNS 53
      • 登记端口号: 1024-49151 为没有熟知端口号的应用程序使用, 需要再IANA登记, 如 Microsoft RDP 3389
      • 短暂端口号: 49152-65535 留给客户进程选择暂时使用
    • 端口号只有本地意义, 即只是为了表示本计算机应用层中的个进程, 不同计算机中的相同端口号没有联系
  • 发送方的复用和接收方的分用

5.5 TCP的拥塞控制

  • 在某段时间, 若对网络中某一资源的需求超过了该资源所能提供的可用部分, 网络性能就要变坏, 这种情况称为拥塞 congestion
    • 网络资源: 网络中的链路容量(带宽), 交换节点中的缓存和处理机等
  • 若出现拥塞而不进行控制, 整个网路的吞吐量将随输入负荷的增大而下降
  • 拥塞控制方法
    • 慢开始 slow start
    • 拥塞避免 congestion avoidance
    • 快重传 fast retransmit
    • 快恢复 fast recovery
  • 流程
    • 发送方维护一个 拥塞窗口 cwnd 的状态变量, 值取决于网络拥塞度, 动态变化
      • 只要没有拥塞, 拥塞窗口就增大, 只要出现, 拥塞窗口就减少
      • 没有按时收到确认报文认定为拥塞 (超时重传)
    • 发送方将拥塞窗口做为发送窗口 swnd swnd = cwnd
    • 维护一个慢开始门限 ssthresh 状态变量
      • cwnd < ssthresh 使用慢开始算法
      • cwnd > ssthresh 停止慢开始算法, 使用拥塞避免算法
      • cwnd = ssthresh 既可以慢开始算法,也可以拥塞避免算法
  • 慢开始 (指开始注入的报文少 不是增长速度慢)
    • 开始为1
    • 指数增加, 2^x
  • 拥塞避免 (使拥塞不容易出现)
    • 每次加1
    • 超时后设置ssthresh拥塞时值的一半
    • 从1开始 慢开始算法
  • TCP Tahoe 只有慢开始和拥塞控制, TCP Reno增加 快重传和快恢复
  • 应为报文丢失,出现超时重传会误判发生拥塞, 降低传输效率
  • 采用快重传, 让发送方今早知道个别报文段的丢失, 尽快发送重传, 不等超时重传计时器超时再重传
    • 接收方不等待自己发送数据是在捎带确认,立刻发送确认
    • 收到失序, 也要立即发出已收到的报文段的重复确认
    • 发送方收到连续3个重复确认, 立即重传
  • 快恢复
    • 一旦收到3个重复确认, 知道只是丢失个别报文段, 不执行慢开始, 而执行快恢复
      • ssthresh和cwnd设为当前的一半, 开始执行拥塞避免算法
      • 另外一种是把快恢复开始时的cwnd值再增大些, 如 ssthresh + 3

5.6 TCP超时重传时间选择

  • 太小,进行不必要的重传, 增大网络负荷. 太大, 降低效率
  • RTO(超市重传时间)应略大于RTT
  • 不能直接使用某次测量的RTT样本计算RTO
  • 计算加权平均往返时间RTTs(平滑的往返时间)
  • 显然RTO应该略大于RTT
  • RFC6298 RTO 计算
  • 无法测准RTT是Karn算法及其改进
  • 出现超时重传, 直接RTO * 2

5.7 TCP可靠传输实现

  • TCP基于以字节为单位的滑动窗口来实现可靠传输
  • 虽然发送窗口时根据接收方的接收窗口试着, 但是同一时刻, 发送方的发送窗口并不总是和接收方的接收窗口一样大
    • 网络传送窗口会滞后
    • 发送方会根据阻塞调整
  • 对于不按序到达的数据应如何处理, TCP无明确规定
    • 如果接收方把不按序列到达的数据丢弃, 简单但浪费资源
    • 先把不按序到达的数据临时存放于接收窗口, 等缺少的收到后再按序交付上层应用进程
  • TCP要求接收方必须有累积确认和捎带确认, 可以减少传输开销
    • 接收方不应该推迟发送确认
      • TCP: 确认推迟的时间不应超过0.5s
  • TCP是全双工通信

    • 收件人方便时可访问
  • 采用客户/服务器方式
  • 三个主要组成构件
    • 用户代理

    • 邮件服务器

    • 定义了许多邮件内容格式
    • 定义了传送编码
    • MIME 也用于 http

    • IMAP Internet Message Access Protocol 因特网邮件访问协议
      • 功能比POP3强大. 游湖可以在自己计算机操控邮件服务器中的邮箱, 因此IMAP是个联机协议
    • 都采用TCP C/S pop3 用熟知端口号110, IMAP熟知端口号 143

    • 暂存又时间限制
